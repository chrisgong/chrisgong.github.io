Android 4.0蓝牙介绍

发表于 2016-03-20   |  

版本支持: 

Android OS API >= 18(4.3)

权限支持: 

<uses-permission android:name="android.permission.BLUETOOTH"/>
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN"/>
<uses-feature android:name="android.hardware.bluetooth_le" android:required="true"/>
如果 android.hardware.bluetooth_le设置为false,可以安装在不支持的设备上使用，判断是否支持蓝牙4.0用以下代码就可以了

流程图:

BluetoothGatt

继承BluetoothProfile，通过BluetoothGatt可以连接设备（connect）,发现服务（discoverServices），并把相应地属性返回到BluetoothGattCallback

BluetoothGattCallback

蓝牙设备操作结果返回。

连接调用流程:

onConnectionStateChange -> onServicesDiscovered()

  1. 此方法会返回指定BluetoothDevice的所有BluetoothGattService对象,通过遍历BluetoothGattService来获取所有BluetoothGattCharacteristic,然后匹配对应的UUID,实例Service,Write,Notify等Characteristic.
  1. 如需开启设备蓝牙通知回传数据,需要往获取的Notify Characteristic的0x2902 BluetoothGattDescriptor中写入BluetoothGattDescriptor.ENABLE_NOTIFICATION_VALUE来开启设备通知服务.

写数据调用流程

写入数据会触发如下方法:

设备接收数据成功后,会已通知形式下发数据,回传如下方法:

BluetoothGattCharacteristic

相当于一个数据类型，它包括一个value和0~n个value的描述（BluetoothGattDescriptor）

BluetoothGattDescriptor

描述符，对Characteristic的描述，包括范围、计量单位等

BluetoothGattService

服务，Characteristic的集合。

BluetoothManager

通过BluetoothManager来获取BluetoothAdapter

BluetoothAdapter

系统蓝牙适配器. 通过此对象可以开启,关闭系统蓝牙;可以开启附近蓝牙设备扫描

LeScanCallback

OS API < 21 蓝牙扫描结果回调类

ScanCallback

OS API >= 21 蓝牙扫描结果回调类

ScanSettings

OS API >= 21 蓝牙扫描设置类

BluetoothProfile

一个通用的规范，按照这个规范来收发数据。
